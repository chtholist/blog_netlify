YAML的语法要求极其严格，可以使用网站检查语法

注意

title: “标题”

date: 创建时间
lastmod: 上次修改时间
draft: 若true，则该文不会被渲染
authors: “[name]”
description: “内容描述”
images: [页面图片]

tags: [标签]
categories: [类别]
series: 系列 series_weight: 在系列的位置 seriesNavigation: 系列导航（T/F）

featuredImage: “文章头图”
featuredImagePreview: “文章预览图”

hiddenFromHomePage: 若true，则不会显示在主页
hiddenFromSearch: 若true，则不会显示在搜索结果

linkToMarkdown: 页脚指向原始md（T/F）

转义字符

在某些特殊情况下文章内容会与 Markdown 的基本或者扩展语法冲突且无法避免.

转义字符语法可以帮助你渲染出想要的内容:

1

{?X} -> X

例如, 两个 : 会启用 emoji 语法，可以像这样使用转义字符语法:

1

{?:}joy:

呈现的输出效果如下:

:joy: 而不是 :joy:

技巧
这个方法可以间接解决一个还未解决的 Hugo 的 issue.

另一个例子是:

1

[link{?]}(#escape-character)

呈现的输出效果如下:

[link](#escape-character) 而不是 link.

音乐

本地音乐
  • root为assets文件夹
1

{{< music url="/music/01.flac" name=歌名（不能有空格） artist=歌手 cover="images\avatar.png" >}}

其他参数

  • mini=false 迷你模式（只有图标）
  • autoplay=false
  • mutex=true 自动暂停其他播放器
  • loop=none 循环播放，参数：all / one / none
  • order=list 播放顺序，参数：list / random
  • list-folded=false 折叠列表
音乐平台

  • server [必需] (第一个位置参数)

    [netease, tencent, kugou, xiami, baidu]

    音乐平台.

  • type [必需] (第二个位置参数)

    [song, playlist, album, search, artist]

    音乐类型.

  • id [必需] (第三个位置参数)

    歌曲 ID, 或者播放列表 ID, 或者专辑 ID, 或者搜索关键词, 或者创作者 ID.
